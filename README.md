# 旅館 客室備品管理（プロトタイプ）

間取り図の上にピンを打ち、各エリアの「セッティングルール」と「置くべき備品」を管理する
シングルページアプリ（SPA）です。バックエンド不要・データはブラウザの **LocalStorage** に保存され、
リロードしても内容は保持されます。

## 技術スタック

- React 18 + TypeScript
- Vite 5
- Tailwind CSS 3
- 永続化：LocalStorage（サーバー不要）

## セットアップ & 起動

```bash
# 1. 依存関係をインストール
npm install

# 2. 開発サーバーを起動
npm run dev
```

起動後、ターミナルに表示される URL（通常は http://localhost:5173 ）をブラウザで開いてください。

### その他のコマンド

```bash
npm run build     # 本番ビルド（dist/ を生成）
npm run preview   # ビルド結果をローカルでプレビュー
```

## 画面と機能

1. **部屋タイプ一覧（トップ）** — 初期データの全6部屋をカード表示。
2. **お部屋の詳細** — 間取り図（画像URL未設定時は青焼き風の簡易図）＋ タップ可能なピン。下部に部屋全体の備品チェックリスト。
3. **エリア別ポップアップ** — ピンをタップすると写真・セッティングルール・紐づく備品を表示。
4. **管理者モード** — ヘッダー右のスイッチで切替。以下を追加・編集・削除でき、即座に LocalStorage へ同期します。
   - 部屋タイプ（部屋名／該当客室番号／間取り図の画像URL）
   - エリア（ピン座標／名称／写真URL／セッティングルール／紐づく備品）
   - 部屋全体の備品チェックリスト

### 管理者モードの操作メモ

- 図面をタップ → その座標に新規ピンを追加
- ピンをドラッグ → 座標を移動（自動保存）／ピンをタップ → そのエリアを編集
- 「初期データに戻す」でシードデータを再投入できます。

## データの保存場所

LocalStorage キー：`ryokan-amenity-data-v1`
初回起動時（キーが空のとき）のみ初期データを自動登録します。データを完全にリセットしたい場合は、
ブラウザの LocalStorage を削除するか、管理者モードの「初期データに戻す」を使用してください。

## IPアドレス制限（Cloudflare Pages）

[functions/_middleware.ts](functions/_middleware.ts) で、指定したIP（CIDR）以外からのアクセスを
403で拒否しています。許可するIPは同ファイル内の `ALLOWED_IPV4_CIDRS` を書き換えてください
（現在はダミー値 `123.45.67.89/32` が入っています）。

```ts
const ALLOWED_IPV4_CIDRS: string[] = [
  '123.45.67.89/32', // ← ここを実際のIPに書き換える
];
```

Cloudflare Pagesはこの `functions/` ディレクトリを自動的にPages Functionsとしてデプロイします。
追加のビルド設定は不要です。

### ダッシュボード側の残骸クリーンアップ（要手動対応）

過去に試した Cloudflare Zero Trust (Access) のポリシー設定は、このリポジトリのコードではなく
Cloudflareアカウントのダッシュボード側にのみ存在するため、コードからは検出・削除できません。
以下を手動で確認してください。

1. **Zero Trust ダッシュボード → Access → Applications**
   このPagesプロジェクトのドメインに紐づくApplicationが残っていないか確認し、不要なら削除する。
2. **Zero Trust ダッシュボード → Access → Policies**
   上記Applicationにひもづく孤立したPolicyがあれば併せて削除する（Applicationを消してもPolicyが
   残るケースがあります）。
3. **Cloudflareダッシュボード → 対象ドメイン → Security → WAF → Custom rules**
   IP制限用に作りかけたルールが残っていないか確認する。今回のPages Functionsによる制限と
   二重・矛盾しないよう、WAF側のIP関連ルールは無効化/削除しておくことを推奨します。
4. **Security → WAF → Tools（旧IP Access Rules）**
   個別に登録したIP許可/拒否ルールが残っていないか確認する。

これらを整理したあとに `functions/_middleware.ts` のIP制限のみが有効な状態になっていることを
確認してください。
